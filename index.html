一、准备工作清单

版本说明：本指南基于 Ubuntu 24.04.4 LTS 和 Gazebo Classic 11，专门针对PX4-ROV（水下机器人）开发环境配置，与PX4 v1.15+版本完全兼容。

硬件要求

  • 内存：至少 8GB RAM（推荐 16GB）
  • 存储：至少 100GB 可用空间
  • CPU：支持虚拟化技术（Intel VT-x / AMD-V）
  • 显卡：支持 OpenGL 3.3+（用于Gazebo仿真）

软件准备清单

  • VMware Workstation Pro 26H1
  • Ubuntu 24.04.4 Desktop AMD64 ISO
  • 网络代理信息（如有）

二、VMware Workstation安装

2.1 下载安装包

访问博通官方下载页面：

https://support.broadcom.com/group/ecx/productfiles?subFamily=VMware%20Workstation%20Pro&displayGroup=VMware%20Workstation%20Pro%2026H1%20for%20Windows&release=26H1&os=&servicePk=543221&language=EN&freeDownloads=true

  • 按照向导完成安装（选择典型安装即可）
  • 安装完成后重启计算机

四、虚拟机创建

4.1 创建步骤

1. 打开 VMware → 创建新虚拟机
2. 选择"典型(推荐)"
3. 选择"安装程序光盘映像文件" → 选择 Ubuntu 24.04.4 ISO
4. 虚拟机名称: Ubuntu-24.04-PX4
5. 位置: 选择有足够空间的磁盘
6. 磁盘容量: 100GB (建议值)
7. 内存: 8192MB (8GB)
8. 处理器: 4核心
9. 网络: NAT模式
10. 完成创建 → 启动虚拟机

4.2 参考视频

详细的虚拟机创建教程：

https://www.bilibili.com/video/BV17i421r7vA/

4.3 Ubuntu安装要点

  • 键盘布局：English (US)
  • 安装类型：Erase disk and install Ubuntu
  • 时区：Asia/Shanghai
  • 用户名：建议英文（如 px4user）
  • 密码：设置简单易记的密码

五、网络连接确认

5.1 网络模式确认

确保虚拟机网络设置为NAT模式：

1. 虚拟机 → 设置 → 网络适配器
2. 确认选择：NAT模式
3. 勾选：连接 → 连接

5.2 网络连通性测试

在Ubuntu终端中测试网络连接：

# 测试网络连接
ping -c 4 baidu.com

# 测试DNS解析
nslookup github.com

# 查看IP地址
ip addr show
问题解决：如果网络不通，请检查VMware NAT服务是否启动，或重启网络服务：sudo systemctl restart NetworkManager

六、网络代理配置

适用场景：如果你的网络环境需要通过代理访问外网，请配置以下代理设置。

6.1 永久代理配置（方式一：/etc/profile）

编辑全局配置文件：

sudo nano /etc/profile

在文件末尾添加（替换为你的代理地址）：

# 永久代理配置（HTTP/HTTPS/SOCKS5 三选一或按需配置）
export http_proxy=http://192.168.1.10:8080
export https_proxy=http://192.168.1.10:8080
export all_proxy=socks5://192.168.1.10:1080
export no_proxy="localhost,127.0.0.1,::1,192.168.0.0/16"

保存并生效：

source /etc/profile

6.2 永久代理配置（方式二：/etc/environment）

编辑系统环境文件：

sudo nano /etc/environment

添加代理信息（格式不同，无需export）：

http_proxy="http://192.168.1.10:8080"
https_proxy="http://192.168.1.10:8080"
all_proxy="socks5://192.168.1.10:1080"
no_proxy="localhost,127.0.0.1,::1,192.168.0.0/16"
提示：方式二需要重启系统或重新登录才能生效，但更稳定。

十、PX4-ROV编译与仿真

10.1 编译PX4-ROV固件

cd ~/uav_workspace/PX4-Autopilot

# 编译支持ROV的SITL版本
make px4_sitl_default

10.2 启动ROV仿真环境

Hippocampus水下机器人仿真：

make px4_sitl gazebo-classic_uuv_hippocampus

通用UUV（无人水下航行器）仿真：

make px4_sitl gazebo-classic_uuv_helica

10.3 ROV仿真控制命令

在 pxh> 提示符下使用以下ROV专用命令：

help                    # 查看所有命令
commander arm          # 解锁ROV
commander disarm       # 锁定ROV
commander takeoff      # 下潜到默认深度
commander land         # 上浮到水面
commander forward 2    # 前进2米
commander backward 1   # 后退1米
commander left 1       # 左移1米
commander right 1      # 右移1米
commander yaw_left 30  # 左转30度
commander yaw_right 30 # 右转30度
commander arm force    # 强制解锁
exit                   # 退出仿真

10.4 ROV仿真参数调整

修改ROV仿真参数：

# 设置水下环境参数
param set SIM_WATER_DENSITY 1025  # 海水密度 kg/m³
param set SIM_WATER_VISCO 0.001   # 水的粘度

# 设置ROV动力学参数
param set UUV_THRUST_SCALE 1.0    # 推力比例
param set UUV_MASS 10.0           # ROV质量 kg

# 保存参数
param save

十二、开发工具安装

12.1 安装 VSCode

sudo snap install --classic code
cd ~/uav_workspace/PX4-Autopilot
code .

12.2 安装 QGroundControl (地面站)

12.2.1 准备工作

# 1. 将当前用户添加到 dialout 组（串口访问权限）
sudo usermod -aG dialout "$(id -un)"

# 2. 禁用 ModemManager（防止其占用串口）
sudo systemctl mask --now ModemManager.service

# 3. 安装 GStreamer（视频流支持）
sudo apt install -y gstreamer1.0-plugins-bad gstreamer1.0-libav gstreamer1.0-gl
sudo apt install -y python3-gi python3-gst-1.0

# 4. 安装 FUSE 和 X11 依赖（AppImage 运行所需）
sudo apt install -y libfuse2 libxcb-xinerama0 libxkbcommon-x11-0 libxcb-cursor-dev
重要： 执行完第 1 步后，需要注销并重新登录才能使权限生效。

12.2.2 下载并运行 QGroundControl

# 创建工作目录并下载
mkdir -p ~/uav_workspace/tools
cd ~/uav_workspace/tools
wget https://d176tv9ibo4jno.cloudfront.net/latest/QGroundControl-x86_64.AppImage

# 赋予执行权限
chmod +x QGroundControl-x86_64.AppImage

# 运行地面站
./QGroundControl-x86_64.AppImage

十五、常见问题与解决方案

15.1 编译错误：子模块无法下载

问题：GitHub 连接不稳定导致子模块下载失败。

解决方案：

# 使用 Gitee 镜像（已在指南中使用）
git clone https://gitee.com/hxlanu/PX4-Autopilot.git --recursive

# 如果仍然失败，尝试单独更新子模块
git submodule sync --recursive
git submodule update --init --recursive --depth=1

15.2 Gazebo-ROV 启动失败

问题：ROV仿真启动时出现错误或模型加载失败。

可能原因及解决方案：

  • 缺少ROV模型：确保PX4版本支持ROV仿真
# 更新子模块，确保包含ROV模型
git submodule update --init --recursive

# 重新安装Gazebo相关包
sudo apt install --reinstall gazebo libgazebo-dev

# 清理并重新编译
make clean
make px4_sitl_default
  • 水下环境参数错误：检查水的密度和粘度设置
# 检查水下环境参数
param show SIM_WATER_DENSITY
param show SIM_WATER_VISCO

# 重新设置正确的参数
param set SIM_WATER_DENSITY 1025
param set SIM_WATER_VISCO 0.001

15.3 ROV控制响应异常

问题：ROV仿真中控制命令无响应或响应异常。

解决方案：

# 检查ROV是否已解锁
commander status

# 检查推力器参数
param show UUV_THRUST_SCALE

# 调整推力器参数
param set UUV_THRUST_SCALE 1.5
param save

# 重启仿真
commander reboot

15.4 水下传感器数据异常

问题：深度传感器、DVL等水下传感器数据不准确。

解决方案：

# 检查传感器参数
param show SENS_DVL_EN
param show SENS_DEPTH_EN

# 启用传感器
param set SENS_DVL_EN 1
param set SENS_DEPTH_EN 1

# 设置传感器噪声参数
param set SENS_DEPTH_NSD 0.01
param set SENS_DVL_NSD 0.05
param save

15.5 Python 依赖冲突

问题：pip 安装时提示 externally-managed-environment 错误。

解决方案：

# 使用 --break-system-packages 标志（已在指南中使用）
pip3 install --break-system-packages <package-name>

# 或创建虚拟环境（推荐）
python3 -m venv ~/px4_venv
source ~/px4_venv/bin/activate
pip install empy numpy toml jinja2 pyyaml

15.6 VMware 虚拟机性能优化

为了让仿真运行更流畅：

  • 启用 3D 加速：虚拟机设置 → 显示器 → 勾选"加速 3D 图形"
  • 分配更多内存：建议 8GB 以上
  • 分配更多 CPU 核心：建议 4 核心
  • 安装 VMware Tools：
sudo apt install -y open-vm-tools open-vm-tools-desktop
